一、数据流图（DFD）概述
================================================================================

数据流图（Data Flow Diagram，DFD）是一种用于表示系统数据流动、处理过程
和数据存储的图形化工具。本文档采用分层DFD方法，从顶层到底层逐步细化。

DFD符号说明：
  [外部实体]    - 系统外部的数据源或数据终点
  (处理过程)    - 对数据进行转换或处理的操作
  {数据存储}    - 数据的静态存储位置
  → 数据流      - 数据在系统中的流动方向

数据流说明：
1. 用户 → SmartDesk系统
   - 用户操作指令（点击、拖拽、输入等）
   - 用户配置信息（主题、路径、账号等）
   - 文件选择和操作请求

2. SmartDesk系统 → 用户
   - 系统响应和状态反馈
   - 文件列表和内容显示
   - 操作结果通知（成功/失败）

3. SmartDesk系统 ↔ 本地文件系统
   - 文件读取请求和文件内容
   - 文件写入操作和确认
   - 目录遍历和文件信息

4. SmartDesk系统 → SMTP邮件服务器
   - 邮件发送请求（包含邮件内容、附件）
   - SMTP认证信息

5. SMTP邮件服务器 → SmartDesk系统
   - 发送状态响应（成功/失败）
   - 错误信息

6. SmartDesk系统 ↔ 云盘服务器
   - 文件上传请求和文件数据
   - 文件下载请求和文件内容
   - 目录操作请求（创建、删除、移动）
   - 云盘文件列表和元数据

主要模块说明：

1. 主窗口控制器 (MainWindow)
   - 功能：协调各功能模块，处理用户输入，管理UI界面
   - 输入：用户操作、配置信息
   - 输出：模块调用指令、UI更新请求

2. 文件系统模块 (FileSystem)
   - 功能：文件浏览、文件操作、标签管理
   - 输入：用户操作、文件路径
   - 输出：文件列表、操作结果

3. 剪贴板模块 (Clipboard)
   - 功能：剪贴板监听、历史记录管理、数据类型识别
   - 输入：系统剪贴板数据
   - 输出：剪贴板历史、复制操作

4. 邮件服务模块 (EmailService)
   - 功能：邮件发送、附件管理、SMTP配置
   - 输入：邮件内容、附件文件
   - 输出：发送状态

5. 云盘模块 (PersonalDrive)
   - 功能：云盘文件管理、上传下载、同步
   - 输入：云盘操作请求
   - 输出：云盘文件列表、同步状态

6. 备份模块 (FileBackup)
   - 功能：文件备份、备份记录管理、文件恢复
   - 输入：备份请求、恢复请求
   - 输出：备份状态、恢复结果

7. 标签日程模块 (TagSchedule)
   - 功能：文件标签管理、日程提醒
   - 输入：标签操作、日程设置
   - 输出：标签列表、提醒通知

8. 数据库服务层 (dbService/dbManager)
   - 功能：统一数据访问接口、数据库连接管理
   - 输入：数据查询、数据更新请求
   - 输出：查询结果、更新确认

   - 标签按钮点击

   - 文件树结构显示
   - 文件标签显示
   - 面包屑路径显示
   - 操作成功/失败提示

3. FileSystem UI层 → QFileSystemModel
   - 文件路径查询请求
   - 目录切换请求
   - 文件信息获取请求

   - 文件树数据（目录结构、文件列表）
   - 文件属性（名称、大小、类型、修改时间）

5. QFileSystemModel → 本地文件系统
   - 目录遍历请求
   - 文件元数据读取

6. 本地文件系统 → QFileSystemModel
   - 目录内容列表
   - 文件属性信息

7. FileSystem 逻辑层 → 本地文件系统
   - 文件读取操作
   - 文件写入操作
   - 文件删除操作
   - 文件移动操作
   - 目录创建操作

8. FileSystem 逻辑层 → dbService
   - 保存文件标签（文件路径、标签内容）
   - 保存备份记录（文件路径、备份时间、备份位置）
   - 查询文件标签
   - 查询备份记录

9. dbService → FileSystem 逻辑层
   - 标签保存确认
   - 备份记录保存确认
   - 标签查询结果
   - 备份记录查询结果

关键数据结构：
- 文件路径 (QString)
- 标签信息 (标签名称、颜色、备注)
- 备份记录 (源路径、目标路径、备份时间、状态)
- 文件属性 (大小、类型、修改时间、权限)

数据流详细说明：

1. 系统剪贴板 → ClipboardMonitor
   - 剪贴板内容变化事件
   - MIME类型信息
   - 数据内容

2. ClipboardMonitor → ClipboardController
   - 新剪贴板数据（原始数据）
   - 数据类型信息
   - 时间戳

3. ClipboardController → ClipboardItemFactory
   - 原始剪贴板数据
   - MIME类型信息

4. ClipboardItemFactory → ClipboardController
   - 创建的ClipboardItem对象
   - Item类型（Text/Image/File）

5. ClipboardController → dbClipboard
   - 保存剪贴板历史记录
   - 数据内容（文本内容、图片数据、文件路径）
   - 时间戳、是否置顶

6. dbClipboard → ClipboardController
   - 保存确认
   - 历史记录查询结果

7. ClipboardController → ClipboardView
   - 添加新Item到列表
   - 移除Item
   - 更新Item状态（置顶/取消置顶）
   - 清空列表

8. ClipboardView → 用户
   - 历史记录列表显示
   - Item预览（文本预览、图片预览、文件信息）

   - 邮件编辑界面显示
   - 附件列表显示
   - SMTP配置界面
   - 发送成功提示
   - 错误信息提示

   - 邮件主题
   - 收件人列表
   - 邮件正文（HTML格式）
   - 附件文件路径列表
   - SMTP配置参数
   - 发送请求

   - 发件人地址
   - 收件人地址列表
   - 邮件主题
   - 邮件正文（MimeHtml对象）
   - 附件列表（MimeAttachment对象）

    - SMTP认证

    - 发送结果（成功/失败）
    - 错误消息

    - 发送成功通知
    - 错误信息显示

  - password: 密码
  - security: 安全连接类型（TCP/SSL/TLS）
- 邮件数据：
  - sender: 发件人地址
  - recipients: 收件人地址列表
  - subject: 邮件主题
  - body: 邮件正文（HTML）
  - attachments: 附件文件路径列表

五、数据流图绘制建议
================================================================================

5.1 绘制工具推荐
- 专业工具：Microsoft Visio, Lucidchart, Draw.io
- 开源工具：Dia, yEd Graph Editor
- 在线工具：ProcessOn, GitMind

5.2 绘制规范
1. 使用标准符号：
   - 圆角矩形或圆形：表示处理过程
   - 矩形：表示外部实体
   - 开口矩形（两条平行线）：表示数据存储
   - 箭头：表示数据流方向

2. 命名规范：
   - 处理过程：使用动词+名词（如"读取文件"、"发送邮件"）
   - 数据流：使用名词短语（如"文件路径"、"邮件内容"）
   - 数据存储：使用名词（如"数据库"、"配置文件"）

3. 层次原则：
   - 每层DFD的输入输出应该与上一层对应
   - 下层DFD是对上层处理的细化
   - 避免跨层数据流

4. 简洁原则：
   - 每个处理过程不超过7个输入输出
   - 数据流名称简明扼要
   - 避免过于复杂的交叉连线

5.3 颜色建议
- 处理过程：蓝色
- 外部实体：绿色
- 数据存储：黄色
- 数据流：黑色或灰色
- 错误流：红色

七、总结
================================================================================

本文档详细描述了SmartDesk项目的数据流图，包括：

1. Level 0 DFD：展示了系统与外部环境的交互
2. Level 1 DFD：展示了系统内部主要模块及其关系
3. Level 2 DFD：详细描述了各模块内部的数据流

关键数据流包括：
- 文件系统：用户操作 → 文件系统模型 → 本地文件系统/数据库
- 剪贴板：系统剪贴板 → 监听器 → 控制器 → 视图层/数据库
- 邮件服务：用户输入 → 邮件构建 → SMTP服务器
- 云盘：用户操作 → 云盘管理器 → 云盘API/本地文件系统/数据库
- 数据库服务：各模块 → 服务接口 → 数据库管理器 → SQLite数据库

通过数据流图，可以清晰地理解SmartDesk系统的数据流动和处理过程，
为系统设计、开发和维护提供重要参考。
